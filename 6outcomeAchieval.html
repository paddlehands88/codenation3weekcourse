Achieving Project Outcomes

When we approach a new project there will be a reason behind this. For my website, acting as a website consultant, the main aim is to attain work. To do this we want any prospective employer / business manager to look at the website, be impressed upon the content, and then make contact whether by email or phone.

The website needs to be accessible to any prospective Employer, so we have to pay special attention to the HTML code used. Search Engine Optimisation (SEO) uses HTML to help determine what rank your page appears when certain keyword searches are carried out. Google, Bing, etc. need to provide the most relevant search results to a user so they find the info they are looking for quickly and easily. We can structure our HTML code in a way to help this process and get further up Google’s or Bing’s search results. Let’s say that we have found the most used search term when someone is looking for a website consultant is ‘digital consultant.’ We can make this the title of our homepage which will raise its ranking for this search.

CSS has been used to create the contact forms at the end of each page. A metric that we would look at would be the number of visitors to the site and the conversion rate of those who used the contact form and made contact (due to their interest in the consulting services.)

Trello is a project management tool. All tasks for a project are categorised into three categories:

  1. To Do - Tasks not yet begun
  2. Doing - Tasks that have begun and are in progress
  3. Done - Completed tasks

These tasks are displayed on a ‘board’ which can be named after the project - for example - Website. A project such as creating a website involves many tasks and this software helps to organise this work. This can then be used to organise a person’s schedule of the work they are to do. Trello boards can also be used by teams with multiple users, and a project manager could use it to check on the status and progression of a project.
